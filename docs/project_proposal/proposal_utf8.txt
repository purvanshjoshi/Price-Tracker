PROPOSAL DESCRIPTION (10 pts)
Motivation (1 pt)
Indian shoppers waste 15-20 minutes manually comparing prices across Amazon, Flipkart, and other platforms before buying. With e-commerce growing 25% yearly, this inefficiency costs users ?500-2000 per purchase in missed deals. Price Scout solves this by automatically scraping prices simultaneously using Java multithreading, delivering the best deal in 1.8 seconds (vs 6+ seconds manually).
Our motivation: Build a production-grade portfolio project that demonstrates core Java syllabus (multithreading, collections, JDBC, Swing) while solving a real problem. Post-evaluation, we'll deploy to Chrome Web Store with affiliate links for monetization-turning academic work into resume gold.?
State of the Art / Current solution (1 pt)
Existing solutions like Google Shopping aggregate via APIs but miss 40% of Indian sites. Browser extensions (Honey, PriceBlink) lack deep Flipkart/Amazon integration and don't store price history. Mobile apps (Pricekart) require manual input and lack desktop support.
Our Edge: Real-time Jsoup web scraping hits actual product pages, MySQL price_history tracks trends, Java Swing dashboard provides analytics. No dependency on limited APIs-direct HTML parsing ensures 95% coverage across 5+ platforms.
Project Goals and Milestones (2 pts)
Primary Goal: Deploy fully-functional price comparison engine by phase 3

Project Approach (3 pts)
Backend (Parth): Spring Boot REST API (/api/compare?product=iPhone). ExecutorService (3 threads) scrapes Amazon/Flipkart simultaneously using Jsoup CSS selectors. TreeMap sorts prices.
Frontend (Purvansh): Java Swing dashboard (JFrame + JTable). HttpClient calls API, org.json parses response, displays comparison table.
Integration (Vansh): MySQL (price_history table), JUnit5 tests, Postman API validation.
Technologies: Java 17, Spring Boot 3.x, Jsoup 1.17, Swing, MySQL 8, Maven.
